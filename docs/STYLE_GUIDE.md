Our documentation should feel like a **DARPA technical briefing** or a **physics paper**, not a marketing landing page.

## 3. Visual Grammar
*   **Headers**: Numbered Sections (1.0, 1.1).
*   **Tables**: Clean, data-heavy. No complex HTML formatting unless necessary for alignment.
*   **Badges**: Monochrome (Black/Gray) only. No colorful "shield walls".
*   **Diagrams**: Mermaid or high-contrast architectural schematics.

## 4. Tone Example
*   *Bad*: "AeonSage is an awesome tool to build agents! It saves you money!"
*   *Good*: "AeonSage functions as a deterministic substrate for autonomous cognition, optimizing token economics via optimistic cascading logic."
